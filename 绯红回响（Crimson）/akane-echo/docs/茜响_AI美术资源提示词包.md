# 🎨 《茜响 / Akane's Echo》AI 美术资源生产提示词包 v3.0

> 配合《茜响》策划案 v2.0 使用。本文档不再把 AI 图像作为单纯参考，而是面向 **Codex 图片生成** 的可生产资产方案：角色锚点图 → 动作帧表 → 地图组件 → 批次验收。

## 📌 生产总原则

### 1. 项目基线

| 项 | 规格 |
|----|------|
| 游戏类型 | 纯 2D 横版侧视类银河城 |
| 引擎基线 | Godot 4.6，2D 节点与贴图资源 |
| 角色资产 | 清线动漫角色，透明背景，偏写实比例 |
| 场景资产 | 低饱和分层水彩横版场景，支持远 / 中 / 近 / 氛围层拆分 |
| 唯一强调色 | 绯红 `#C8102E` |
| 禁止项 | 2.5D 俯视、3D 塑料质感、摄影写实、霓虹杂色、无意义红色 |

### 2. 内部风格基线提示词

所有资源生成都必须带上这段基线，并按具体条目补充主体与用途：

```text
《茜响 / Akane's Echo》纯 2D 横版游戏美术，清线动漫角色与低饱和分层水彩场景结合，
淡灰、米白、浅青、低饱和区域色为主，唯一高饱和强调色只能是绯红 #C8102E，
柔和水彩笔触，纸张颗粒纹理，轻微胶片颗粒，忧郁安静，柔和漫射光，
角色轮廓清楚，动作剪影可读，场景前景 / 中景 / 远景层次明确，
专业 2D 游戏资产，适合 Godot 横版侧视项目使用。
```

### 3. 通用负面约束

```text
不要 2.5D 俯视，不要 3D 渲染，不要摄影写实，不要塑料质感，不要 Q 版，不要夸张可爱风，
不要高饱和蓝、绿、黄、紫、橙等第二强调色，不要霓虹灯，不要彩虹色，不要复杂彩色边光，
不要文字、水印、签名、logo、界面截图，不要多余角色，不要现代城市物件，
不要错误红色用途：咏身上不能有红色，普通敌人除弱点外不能有红色，场景红色面积不能超过 5%。
```

### 4. 资产清单字段

| 字段 | 填写方式 |
|------|----------|
| `asset_id` | 唯一编号，例如 `AKN_REF_front_01`、`MAP_Forest_key_01` |
| `类别` | 角色 / 动作 / Boss / 敌人 / 地图 / 组件 / UI |
| `用途` | 游戏内用途或生产用途 |
| `视角/比例` | 侧视、正面、背面、16:9、1:1、透明背景等 |
| `红色锚点` | 红色出现的位置；没有红色写"无" |
| `生成提示词` | 可直接交给 Codex 图片生成 |
| `负面约束` | 在通用负面约束基础上补充 |
| `验收标准` | 入库判定 |
| `关联策划案章节` | 用于追溯来源 |

### 5. 批次命名约定

```text
AKN = 阿茜
YNG = 咏
BOS = Boss
ENM = 普通敌人
ELT = 精英敌人
MAP = 地图 key art / 房间原型
PAR = 视差层
PRP = 场景组件 / 道具
UI  = UI 视觉资源
```
